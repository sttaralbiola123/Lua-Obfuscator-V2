Instructions
How It Works
Your Lua code is parsed into an AST (Abstract Syntax Tree)
Compiled into custom bytecode with randomized opcodes
Constants are multi-layer encrypted with unique keys
A unique VM interpreter is generated to run the bytecode
Junk instructions are injected to confuse analysis tools
Protection Layers
Random opcodes — different every compile
Multi-key XOR + byte shuffle encryption
Custom base alphabet (replaces standard base64)
~45% junk instruction injection
Opaque predicates + dead code branches
All VM internals use randomized variable names
Limitations
Complex closures / upvalues may not fully execute
pcall/xpcall behavior may differ inside VM
Max script size: 50KB
Always test your obfuscated script before use
